DBMS & RDBMS

DBMS: DataBase Management System

It is a software that is used to store, manage and retrive data efficiently.

It allows users and applications to:

  • Store Data
  • Read Data
  • Update Data
  • Delete Data

Examples:

  • MySQL
  • Oracle
  • MongoDB
  • PostgreSQL

Example:


{
    "name": "Luka",
    "age": 17,
    "skills": ["JavaScript", "Python"]
}

RDBMS: Relational Database Management System

It is a type of DBMS that stores data in tables and connects these tables using relationship

It allows users and applications to:

  • Organizes data into tables
  • Uses primary keys and foreign keys
  • Maintains relationships between tables
  • Uses SQL to manage data

Primary Keys: unique identifier for a row

Foreign keys: links one table to another

Examples:

  • MySQL
  • Oracle
  • PostgreSQL
  • SQL Server

Example:

id(PK) name age
1 Luka 17
id(PK) user_id(FK) skill
1 1 JavaScript
2 1 Python

RDBMS is same as DBMS but it uses tables and relations between tables.

Relational Databases & Irrational Databases(NoSQL)

Relational Databases

  1. Tables: Data is stored in rows(records) and columns(fields).

  2. relationships:
    • One-to-One → One row in table A matches one row in table B
      • Example: Each person has one passport
    • One-to-Many → One row in table A matches many rows in table B
      • Example: One user can have many posts
    • Many-to-Many → Many rows in table A match many rows in table B
      • Example: Students ↔ Courses (students can take multiple courses, each course has many students)

  3. Schema: RDBs have a fixed structure (columns must be defined in advance).

  4. SQL: You use SQL to create, read, update, and delete data.

  5. ACID Properties: (ACID = rules that make database transactions safe and reliable.)
    • Atomicity: Everything in a transaction happens or nothing happens
      • Example: Transfer $10 → subtract $10 from A and add $10 to B. If one fails, nothing happens.
    • Consistency: Data stays correct
      • Example: Age cannot be negative. DB ensures this rule.
    • Isolation: Transactions don't interfere
      • Example: Two users updating the same row at the same time won't break data.
    • Durability: Once saved, data is permanent
      • Example: Your bank balance is safe even if the system crashes after the transaction.

Irrational Databases

  1. No tables: Stores data as documents, key-value pairs, graphs, or wide-column stores.
  2. Flexible structure: Each record(row) can have different fields(column).
  3. Scales easily: Handles huge amounts of data and high traffic.
  4. No strict relationships: Tables (or documents) don't need to be linked.
  5. Query languages differ: MongoDB uses MQL(MongoDB Query Language), others have their own syntax.

SQL

SQL = Structured Query Language

  • It is the language used to talk to relational databases(RDBMS).
  • You use it to store, read, update, and delete data.

  • Example: MySQL, Oracle, SQL Server → all use SQL.
