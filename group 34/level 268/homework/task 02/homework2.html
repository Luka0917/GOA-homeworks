Radix Sort

A non-comparative sorting algorithm that sorts numbers digit by digit using Counting Sort as a helper at each step

Key Terms

  • LSD (Least Significant Digit) - sorts right to left (ones -> tens -> hundreds) used for integers
  • MSD (Most Significant Digit) - sorts left to right better for strings
  • Stable sort - equal elements keep their original relative order
  • Non-comparative - never compares two elements directly groups by digit only
  • k - number of digits in the largest number (802 -> k = 3)

How It Works

  1. Find the largest number to know how many passes are needed (k)
  2. Sort all numbers by their ones digit using counting sort
  3. Repeat for tens then hundreds and so on
  4. After all k passes the array is fully sorted

Time & Space Complexity

Time: O(n * k)

Space: O(n + k)

n = number of elements | k = number of digits in the largest number

Pros

  • Faster than O(n log n) sorts when k is small
  • Stable sort
  • No element comparisons

Cons

  • Needs extra memory — not in-place
  • Slower when k is large
  • Doesn't work directly on floats or negative numbers
  • Only good for integers or fixed-length strings
